WEB-SHOOTER

documentation crawler · spins any docs site into AI context
WHAT IS IT?

Turn any docs site into clean Markdown

WEB-SHOOTER is a small self-hosted service that crawls a documentation website (Sphinx, MkDocs, Docusaurus, Read-the-Docs, generic HTML) and converts it into clean Markdown — either one big file for an LLM context window, or one file per page for RAG.

①

Single .md

One Markdown doc with TOC. Paste into ChatGPT / Claude / Gemini context.

②

Split folder

One file per page + index.md. Drop into a RAG pipeline or NotebookLM.

③

Web UI + API + CLI

FastAPI service with a retro web UI and JSON endpoints. Plus a CLI for scripts.

④

One-line docker

Stateless container. No host bind-mounts required. Artifacts in a named volume.

⬢ Source on GitHub ⬇ Self-host in 30 seconds 📖 Setup guide
WHY NO PUBLIC DEMO?

This page is a landing, not a live scraper

WEB-SHOOTER needs a server: it has to make HTTP requests out to other docs sites, parse them, write files to disk, and serve downloads. A browser-only page can't do any of that — CORS would block almost every request, and there's nowhere to put the result.

Running a free public instance also means anyone could point it at anything (rate-limits, abuse, hosting costs, legal liability for whatever they crawl). So: you host it yourself, on your machine, in 30 seconds. The code is MIT-licensed.

Be a good citizen: respect target sites' robots.txt, throttle your crawls (the default 0.2s delay is a floor, not a ceiling), and don't redistribute scraped content in ways its license doesn't allow.
SELF-HOST IN 30 SECONDS

Docker (recommended)

git clone https://github.com/KeeperOfStack/web-shooter.git
cd web-shooter
docker compose up -d --build

# UI:   http://localhost:8088/
# API:  http://localhost:8088/docs
# health: http://localhost:8088/healthz

Or run it bare (Python 3.12)

python3.12 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn server:app --host 0.0.0.0 --port 8088

Or use the CLI (no server)

# one big .md
python docscrape.py https://docs.example.com/ -o example.md

# one file per page, plus a .zip
python docscrape.py https://docs.example.com/ --split --zip -o example_docs
HTTP API

Scriptable from anywhere

# kick off a job
curl -s -X POST http://localhost:8088/scrape \
  -H 'content-type: application/json' \
  -d '{"url":"https://docs.example.com/","max":300,"delay":0.2,"mode":"single"}'
# → {"job_id":"abc123..."}

# poll
curl -s http://localhost:8088/jobs/abc123...

# download when status is "complete"
curl -OJ http://localhost:8088/jobs/abc123.../download

Full route list lives at /docs on a running instance (FastAPI auto-generated OpenAPI).

UNDER THE HOOD

How it works

  • Crawler: breadth-first, same-path-scope only (a crawl of /docs/ won't wander into /blog/).
  • Content extraction: BeautifulSoup + a curated list of MAIN_SELECTORS / STRIP_SELECTORS tuned for Sphinx, MkDocs, Docusaurus, Read-the-Docs themes.
  • Markdown: markdownify preserves fenced code blocks and tables (html2text mangles them).
  • Limits: requests-only (no JS rendering), hard cap of 2000 pages per job, configurable delay between requests.

Stack: Python 3.12 · FastAPI · BeautifulSoup4 + lxml · markdownify · requests. ~300 LOC of crawler.
